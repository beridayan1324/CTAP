# פרק א' – ייזום {#פרק-א-ייזום}

# תיאור ראשוני של המערכת {#תיאור-ראשוני-של-המערכת}

**תקציר כולל ורציונל הפרויקט**

בעולם של היום, תקשורת בין-אנושית עברה בחלקה הגדול למרחב הדיגיטלי. אפליקציות צ'אט, פלטפורמות שיתוף מסמכים, וכלי שיחה מרחוק הפכו לנורמה. ואולם, רוב הפתרונות הקיימים עוצבו עבור אדם הממוצע היושב מול מקלדת סטנדרטית. בסביבות שבהן הגישה למקלדת מוגבלת — בין אם בשל מגבלה פיזית, סביבת עבודה ייחודית, או צורך בהקלדה שקטה ולא גלויה — אין כמעט פתרון מובנה שמחבר בין חומרה לא-סטנדרטית לבין פלטפורמת תקשורת מאובטחת.

המערכת שפיתחתי, CTAP (Communication Transmitting Analog Protocol), באה לתת מענה לפער הזה. CTAP היא פלטפורמת תקשורת היברידית המורכבת משתי שכבות: שכבת החומרה — כפפה הצמודה ליד המשתמש ומצוידת בחיישני לחץ (FSR) על כל אצבע — ושכבת התוכנה — שרת צ'אט מבוסס WebSockets, עם הצפנה מקצה לקצה, אימות קריפטוגרפי, וניהול יומן בקרה עצמאי. לחיצה על אצבע אחת או יותר מייצרת שילוב בינארי של חמישה סיביות, שמומר לאות לפי מילון קבוע ומשודר לשרת כהודעת טקסט מוצפנת בזמן אמת.

**מה המוצר המוגמר אמור לבצע**

המוצר המוגמר אמור לאפשר לכל משתמש להירשם ולהתחבר לפלטפורמת הצ'אט, להצטרף לחדרי שיחה וירטואליים, ולשלוח הודעות הן דרך ממשק Web רגיל והן דרך הכפפה החומרתית. הכפפה מורכבת מרכיב ESP32 ומחמישה חיישני FSR, אחד לכל אצבע. כאשר המשתמש לוחץ על צירוף אצבעות מסוים, המיקרו-בקר מחשב את השילוב הבינארי, ממירו לאות לפי מילון קבוע, ושולח את ההודעה הסופית דרך חיבור USB טורי אל הדפדפן. הדפדפן, בתורו, קורא את הנתונים בעזרת Web Serial API, מצפין את הטקסט באמצעות AES-256-GCM, ומשדר אותו לשרת Flask דרך WebSocket. השרת מאמת את זהות הלקוח באמצעות לחיצת יד קריפטוגרפית, מפצח את ההצפנה, ומפיץ את ההודעה לכל המשתמשים בחדר. כל אירוע — חיבור, אימות, הודעה — נרשם ביומן Audit נפרד הנגיש לניהול ולבקרה.

**למה בחרתי את הפרויקט ואתגרים צפויים**

בחרתי את הפרויקט הזה מפני שהוא משלב שני עולמות שמעניינים אותי: חומרה ותוכנה. רציתי לבנות משהו שמרגיש פיזי ומוחשי, לא רק קוד שרץ בשקט ברקע. הרעיון של כפפה שמתרגמת תנועות אצבע להודעות טקסט נשמע לי כאתגר טכני אמיתי שאאלץ ללמוד ממנו הרבה, ובמיוחד תחומים שלא נגעתי בהם קודם לכן, כמו קריאת אותות אנלוגיים ואבטחת WebSockets.

האתגרים שזיהיתי מראש היו:

1. סנכרון בין קריאות האנלוג של חיישני ה-FSR לבין לוגיקת הזיהוי — חיישן FSR יכול לייצר רעשים חשמליים שמפוענחים בטעות כלחיצה.
2. הבטחת קצב תקשורת נמוך מספיק כדי שהשרת לא יחנוק את התעבורה, אך גבוה מספיק כדי שהמשתמש ירגיש שהמערכת מגיבה בזמן אמת.
3. הטמעת אבטחה מקצה לקצה מבלי לפגוע בביצועים — הצפנת AES-256-GCM, אימות JWT, ולחיצת יד קריפטוגרפית מוסיפים עומס חישובי שצריך לנהל בזהירות.
4. Web Serial API — טכנולוגיה חדשה יחסית, התלויה בדפדפן ואינה נתמכת בכל הסביבות, מה שמגביל את תאימות המערכת.

# הגדרת הלקוח {#הגדרת-הלקוח}

**למי מיועדת המערכת ומי הולך להשתמש בה**

המערכת מיועדת לשלושה סוגי משתמשים:

1. **מפעילי חומרה** — אנשים המעוניינים להשתמש בשיטת קלט חלופית לתקשורת. לדוגמה, אנשים עם מגבלה פיזית המקשה על שימוש במקלדת רגילה, אנשי שטח שידיהם תפוסות, או משתמשים הזקוקים להקלדה שקטה ולא גלויה. המפעיל מחבר את הכפפה לשקע ה-USB של המחשב, פותח את הדפדפן, ולוחץ על כפתור חיבור ה-Serial כדי לאפשר קריאת נתוני הכפפה.
2. **משתמשי Web** — כל משתמש המחובר לפלטפורמת הצ'אט דרך הדפדפן, ללא חומרה נוספת. הוא יכול לנהל שיחות בחדרים וירטואליים, לראות הודעות של כל המשתתפים בחדר, ולשלוח הודעות דרך ממשק הקלדה רגיל.
3. **מנהלי מערכת וצוותי בקרה** — אנשים האחראים לניטור תקינות המערכת, הרשאות גישה, ובדיקת יומני האירועים. הם יכולים לגשת למסך ה-Audit שבו מוצגות כל ההתחברויות, ההתנתקויות ואירועי האימות, וכן להתחבר ישירות לשרת ה-TCP החיצוני לשליפת נתונים גולמיים.

# הגדרת יעדים/מטרות {#הגדרת-יעדיםמטרות}

**מה המטרות המרכזיות של המערכת המוצעת**

המטרות המרכזיות של המערכת הן:

1. **אבטחה מקצה לקצה** — כל הודעה שנשלחת מהכפפה מוצפנת עוד לפני שיוצאת מהדפדפן, באמצעות AES-256-GCM. אימות הלקוחות מתבצע בלחיצת יד קריפטוגרפית המבוססת על SHA-256 ועל סוד משותף, ומניעת ניחוש סיסמאות מטופלת על ידי Rate Limiter ייעודי.
2. **תקשורת מהירה ובזמן אמת** — שימוש בפרוטוקול WebSocket מאפשר דו-סטריות מלאה בין הלקוח לשרת ללא עיכוב ניכר, בניגוד ל-HTTP polling שגורם לתחושת האטה.
3. **קריאת נתוני חיישנים ברזולוציה גבוהה** — ה-ESP32 מצויד ב-ADC בעל רזולוציה של 12 ביט, ויחד עם מנגנוני debounce ו-Threshold Hysteresis בקוד הפירמוור, מבטיח שרק לחיצות אמיתיות יירשמו ולא רעשים חשמליים.
4. **ניהול יומן שקוף ועצמאי** — שרת ה-Audit רץ בתהליך TCP נפרד, ולכן רישום האירועים אינו מאט את זרימת הודעות הצ'אט בשרת הראשי.
5. **תמיכה בריבוי חדרים** — המערכת מנהלת Room Registry דינמי המאפשר למשתמשים לעבור בין חדרי שיחה ללא צורך בהתחברות מחדש.

# בעיות, תועלות וחסכונות {#בעיות-תועלות-וחסכונות}

**מה הבעיות אותן אנו מנסים לפתור**

ישנן שתי בעיות מרכזיות שהמערכת מתמודדת איתן:

הבעיה הראשונה היא **חשיפת תעבורת IoT לתקיפות**. אפליקציות IoT רבות שולחות נתונים מחיישנים ישירות לרשת ללא הצפנה ולעיתים ללא אימות. במצב כזה, תוקף המצוי באותה הרשת יכול ליירט, לשנות, או לזייף הודעות ללא מאמץ. בעיה זו בולטת במיוחד כאשר ה-ESP32 שולח נתונים גולמיים דרך Wi-Fi ישירות לשרת מרוחק.

הבעיה השנייה היא **הקושי בהקלדה בסביבות לא-סטנדרטיות**. אנשים הנמצאים בסביבות שבהן שימוש במקלדת רגילה אינו אפשרי — בין אם בשל מגבלה פיזית, סביבת עבודה מיוחדת, או ציוד מגביל — לא יכולים להשתמש ברוב כלי התקשורת הנפוצים. אין פתרון מסחרי קיים שמאפשר חיבור ישיר של חיישני לחץ לפלטפורמת שיחה מאובטחת.

**מה התועלות שסביר לצפות מהמערכת**

עם המערכת הזו, כל הודעה שנשלחת מהכפפה עוברת הצפנה עוד לפני שיצאה מהדפדפן, כך שגם אם התעבורה נתפסת ברשת, לתוקף אין יכולת לקרוא את תוכנה. בנוסף, אימות הלקוח בלחיצת יד SHA-256 מבטיח שרק לקוח שיודע את הסוד המשותף יוכל להתחבר לשרת, ומניעת Brute Force דרך ה-Rate Limiter מבטיחה שניסיונות ניחוש חוזרים ייחסמו אוטומטית לאחר 12 כשלונות בחלון של 5 דקות.

מצד החומרה, מי שמשתמש בכפפה מקבל שיטת קלט שקטה, לא גלויה, וניתנת לשימוש כאשר הידיים עסוקות — יכולת שאינה קיימת בשום פלטפורמת צ'אט מסחרית קיימת.

**אילו שירותים המערכת תיתן**

המערכת תיתן את השירותים הבאים:

- ניהול משתמשים: הרשמה, התחברות, אימות JWT, וניתוק מאובטח.
- ניהול חדרים וירטואליים: יצירה דינמית, הצטרפות ועזיבה של חדרי שיחה.
- שידור הודעות בזמן אמת לכל המשתתפים בחדר.
- קבלת קלט מהכפפה החומרתית דרך Web Serial ושידורו לשרת כהודעה מוצפנת.
- יומני Audit מרוכזים ומאובטחים, הנגישים דרך ממשק Web וגם דרך שרת TCP חיצוני.

**השוואת העבודה עם פתרונות ויישומים קיימים**

**Slack** — פלטפורמת צ'אט מקצועית עם תמיכה בערוצים, הודעות ישירות, ושיתוף קבצים. Slack אינה תומכת בשום צורה של קלט חומרתי ישיר דרך Web Serial, ואין בה אפשרות לחבר כפפה או חיישני לחץ לשרת השיחה. [קישור לאתר](https://slack.com)

**WhatsApp Web** — פלטפורמת הודעות נפוצה עם הצפנה מקצה לקצה. ממשק הקלדה בלבד, ואין תמיכה ב-Web Serial API ולא ביכולת לחבר התקן חומרה לפלטפורמה. [קישור לאתר](https://web.whatsapp.com)

ההבדל המהותי של CTAP מהפתרונות הקיימים הוא שהיא היחידה המשלבת את שכבת החומרה (חיישני FSR, ESP32) עם שכבת הצ'אט המוצפן כמערכת אחידה אחת, ומאפשרת לשניהם לתקשר ישירות דרך Web Serial API ו-WebSockets מאובטחים.

# האם צפויים קשיים או מגבלות בהגדרת המערכת {#האם-צפויים-קשיים-או-מגבלות-בהגדרת-המערכת}

**האם מדובר בטכנולוגיה חדשה ובלתי מוכרת**

שכבת החומרה — ESP32 ו-Arduino — היא טכנולוגיה מבוססת ומתועדת היטב. הבעיה העיקרית מבחינת היכרות עם הטכנולוגיה נוגעת ל-Web Serial API. מדובר בתקן חדש יחסית שנתמך רק בדפדפני Chromium (Chrome ו-Edge) ואינו זמין ב-Firefox ו-Safari. הכתיבה ב-JavaScript עם API אסינכרוני כמו `navigator.serial.requestPort()` ולולאת `reader.read()` הייתה תחום חדש עבורי שלמדתי במהלך הפרויקט.

בנוסף, הצפנת AES-256-GCM וניהול נונס (nonce) לא היו מוכרים לי לפני הפרויקט, ואלה חייבו לימוד עצמאי של ספריית `cryptography` ב-Python ושל `crypto.subtle` ב-JavaScript.

**האם קיימים סייגים בהגדרת המערכת**

קיים סייג חשוב הנוגע לקצב ההקלדה דרך הכפפה. חלון ההמתנה לאיסוף לחיצות האצבעות נקבע על 300ms — כלומר כל אות דורשת מינימום 300 אלפיות שניה להקלדה. המשמעות היא שקצב ההקלדה המקסימלי דרך הכפפה מוגבל לכ-200 אותיות לדקה, בעוד שמקלדת רגילה יכולה להגיע ל-400 אותיות לדקה ויותר. זהו סייג מובנה שנובע מהצורך לאפשר למשתמש להשלים את לחיצותיו לפני שהמערכת מפענחת את האות.

סייג נוסף הוא תלות בדפדפן — Web Serial API אינו פועל ב-Firefox ו-Safari. משתמשים באותם דפדפנים לא יוכלו לחבר את הכפפה, אך יוכלו עדיין להשתמש בממשק הצ'אט הרגיל.

# תיחום הפרויקט {#תיחום-הפרויקט}

**התחומים בהם הפרויקט עוסק**

הפרויקט עוסק במספר תחומים טכנולוגיים במקביל:

- **רשתות ותקשורת מחשבים** — מימוש שרת WebSocket מרובה לקוחות, ניהול חדרים וירטואליים, ותקשורת TCP עצמאית לשרת ה-Audit.
- **פיתוח חומרה ותוכנה משובצת (Embedded Systems)** — תכנות ESP32 ב-C++ עם ניהול ADC, debounce ו-hysteresis ברמת הפירמוור.
- **אבטחת מידע והצפנה** — שימוש ב-AES-256-GCM להצפנת הודעות, bcrypt לאחסון בטוח של סיסמאות, JWT לאימות, ולחיצת יד קריפטוגרפית המבוססת SHA-256 עם `hmac.compare_digest`.
- **בניית מסדי נתונים רלציוניים** — עיצוב סכמת SQLite עם שלוש טבלאות, שימוש ב-WAL mode לשיפור ביצועי כתיבה מקבילה, וניהול גישה thread-safe באמצעות mutex.
- **פיתוח Web Frontend** — בניית ממשק משתמש בדפדפן עם JavaScript, HTML ו-CSS, תוך שימוש ב-Web Serial API ו-WebSocket API.

# פירוט המערכת {#פירוט-המערכת}

בעשורים האחרונים הפכה תקשורת מבוססת רשת לבסיס של חיי היומיום — ממשלוח הודעות פרטיות ועד ניהול פגישות עבודה. ואולם, מרבית הפלטפורמות הקיימות מניחות מראש שהמשתמש יושב מול מקלדת ומסך. CTAP יוצאת מנקודת מוצא שונה: היא מאפשרת לאדם לשלוח הודעות טקסט דרך לחיצות אצבע, ולחבר כלי חומרה פיזי ישירות לממשק הצ'אט המאובטח.

המערכת מורכבת מארבע שכבות עיקריות הפועלות יחד. השכבה הראשונה היא יחידת הקצה — כפפת ESP32 המחוברת ל-USB של המחשב. כל אצבע מחוברת לחיישן FSR (Force Sensitive Resistor) המתרגם לחץ לערך אנלוגי. הפירמוור שכתבתי ב-C++ קורא את הערכים האנלוגיים כל 5ms, מיישם מנגנון debounce ו-Threshold Hysteresis למניעת רעשים, ולאחר חלון של 300ms מרכיב את שילוב האצבעות לאות בינארית ומדפיס אותה לפורט הטורי.

השכבה השנייה היא תיווך הדפדפן. קוד ה-JavaScript בדף הצ'אט קורא את הנתונים מהפורט הטורי בעזרת Web Serial API, מקפל כל אות לתוך הודעת JSON, מצפין את הטקסט עם AES-256-GCM, ומשדר אותו לשרת דרך WebSocket.

השכבה השלישית היא שרת Flask המאזין על פורט 8766. השרת מוודא שכל לקוח מחובר עבר לחיצת יד קריפטוגרפית, מנהל Room Registry המאפשר ריבוי חדרים, מפצח את ההצפנה של כל הודעה מהכפפה, ומפיץ את הטקסט הגלוי לכל המחוברים לאותו חדר.

השכבה הרביעית היא מסד הנתונים ויומן הבקרה. כל הודעה, חיבור, וניתוק נרשמים ב-SQLite הפועלת ב-WAL mode. רישום זה מתבצע דרך שרת TCP עצמאי הפועל על פורט 8767 ומאפשר שליפת נתוני בקרה ממערכות חיצוניות ללא צורך בגישה ישירה לקובץ מסד הנתונים.

# יכולות המערכת {#יכולות-המערכת}

1. קריאת לחיצות חמש אצבעות מחיישני FSR, המרתן לשילוב בינארי, ופיענוח הרצף לאות לפי מילון קבוע של 26 אותיות.
2. שליחת תו "רווח" על ידי לחיצה בו-זמנית על כל חמש האצבעות, ושליחת ההודעה הסופית לשרת על ידי שתי לחיצות עוקבות של כל חמש האצבעות.
3. העברת נתונים ישירה מהכפפה לדפדפן דרך Web Serial API (USB טורי, קצב 115200 baud).
4. ביצוע לחיצת יד קריפטוגרפית מול השרת עם כל חיבור WebSocket חדש, המבוססת על SHA-256(challenge + shared_secret) עם השוואה constant-time.
5. הצפנת תוכן ההודעות מהכפפה באמצעות AES-256-GCM לפני שליחתן ברשת.
6. ניהול חדרי שיחה וירטואליים — יצירה דינמית, הצטרפות ועזיבה ללא ניתוק מהשרת.
7. אימות משתמשים בשיטת JWT (HS256) עם תוקף של 24 שעות לכל token.
8. מניעת Brute Force בנקודת הכניסה — חסימת IP לאחר 12 כשלונות בחלון של 5 דקות.
9. רישום יומן Audit לכל אירוע (חיבור, אימות, הודעה) ב-SQLite, הנגיש דרך ממשק Web ודרך שרת TCP נפרד.
10. הגנת HTTP Headers — הגדרת `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy` ו-`Permissions-Policy` בכל תגובת שרת.

# יכולות המערכת {#יכולות-המערכת-1}

1. **קריאת נתוני חיישנים (FSR) והמרתם לטקסט בינארי**

**מהות היכולת**

היכולת מאפשרת לכפפת ה-ESP32 לזהות לחיצות אצבעות ולתרגם כל שילוב ייחודי של חמש אצבעות לאות לטין אחת.

**תהליכים ויכולות**

* קריאת ערכי ADC מחמישה פינים אנלוגיים (34, 35, 32, 33, 25) כל 5ms
* השוואת ערך כל חיישן לספי ה-Threshold Hysteresis — לחיצה מעל 500, שחרור מתחת ל-300
* החלת debounce של 50ms למניעת רישום כפול של אותה לחיצה
* פתיחת חלון זמן של 300ms מרגע הלחיצה הראשונה באותה אות
* בתום החלון — בניית מחרוזת בינארית בת חמישה סיביות לפי מצב האצבעות
* שליחת המחרוזת לפונקציית `getLetterFromBits()` המחזירה את האות המתאימה
* הצטברות האותיות במחרוזת `textSentence` עד לשליחה
* לחיצה כפולה על כל חמש האצבעות ("11111" פעמיים ברצף) — שליחת ה-`textSentence` לפורט הטורי וניקויה
* לחיצה חד-פעמית על כל חמש האצבעות ("11111") — הוספת רווח

**אובייקטים בהם היכולת עושה שימוש**

* חיישני FSR (חומרה)
* מיקרו-בקר ESP32 ופינים אנלוגיים
* פורט טורי (Serial, 115200 baud)
* מחרוזת `textSentence` (מצב מצטבר)
* מילון האותיות (`getLetterFromBits`)

---

2. **העברת נתונים ישירה מחומרה לדפדפן (Web Serial)**

**מהות היכולת**

היכולת מחברת בין יחידת הקצה החומרתית (ESP32) לבין הדפדפן, ומאפשרת קריאה אסינכרונית של הנתונים הטוריים ישירות בממשק ה-Web, ללא תוכנת ביניים.

**תהליכים ויכולות**

* לחיצת המשתמש על כפתור "Connect ESP32" בממשק
* קריאה ל-`navigator.serial.requestPort()` לבחירת פורט על ידי המשתמש
* פתיחת הפורט ב-`port.open({ baudRate: 115200 })`
* הפעלת לולאת `readSerial()` — לולאה אסינכרונית הקוראת chunks בעזרת `reader.read()`
* פיצול כל chunk לשורות לפי `'\n'`
* סינון שורות ריקות ושורות מטה-נתונים (כגון `"CTAP FSR 5-Finger Mode Ready"`)
* שליחת כל שורת טקסט תקינה לשרת כהודעת `web_msg` דרך ה-WebSocket הפתוח
* עדכון מצב ה-Serial בממשק (`Connected` / `Offline`) לפידבק ויזואלי למשתמש
* ניתוק מסודר בקריאה ל-`disconnectSerial()` — ביטול ה-Reader וסגירת הפורט

3. **ביצוע Handshake קריפטוגרפי ואימות לקוחות**

**מהות היכולת**

היכולת מבטיחה שכל לקוח המתחבר לשרת דרך WebSocket מוכיח שהוא יודע את הסוד המשותף, עוד לפני שיוכל לשלוח הודעות כלשהן לחדרי השיחה.

**תהליכים ויכולות**

* עם כל חיבור WebSocket חדש — השרת יוצר מחרוזת אקראית בת 16 bytes (`os.urandom(16).hex()`) ושולח אותה ללקוח כ-`auth_challenge`
* הלקוח (בדפדפן) מחשב `SHA-256(challenge + HANDSHAKE_SECRET)` בעזרת `crypto.subtle.digest` ושולח בחזרה את ה-hash בפורמט hex כ-`auth_response`
* השרת מחשב באופן עצמאי את ה-hash הצפוי ומשווה אותו לתגובת הלקוח בעזרת `hmac.compare_digest` — השוואה constant-time שאינה חושפת מידע על הסוד מזמן הביצוע
* אם ה-hash תואם — השרת שולח `auth_result: OK` ורושם `AUTH_SUCCESS` ביומן, מוסיף את הלקוח לחדר `default`
* אם ה-hash אינו תואם, חסר, או בפורמט שגוי (לא 64 תווים hex) — השרת שולח `auth_result: FAIL`, רושם `AUTH_FAIL`, וסוגר את החיבור
* כל הלקוח שלא שלח תגובה תוך 5 שניות — מנותק אוטומטית

**אובייקטים בהם היכולת עושה שימוש**

* WebSocket (תקשורת)

קטע הקוד השלישי: **פענוח AES-256-GCM** (`crypto_utils.py`, שורות 26–33)

```python
def decrypt_payload(self, payload: dict) -> str:
    try:
        nonce = base64.b64decode(payload["nonce"])
        ciphertext = base64.b64decode(payload["ciphertext"])
        plaintext_bytes = self._aesgcm.decrypt(nonce, ciphertext, None)
        return plaintext_bytes.decode("utf-8")
    except Exception as e:
        return f"[DECRYPTION FAILED] {e}"
```

הפונקציה מקבלת dict עם שני שדות base64: `nonce` ו-`ciphertext`. ב-AES-256-GCM, ה-nonce הוא מחרוזת אקראית ייחודית לכל הודעה (12 bytes בדרך כלל) — חיוני שלא יחזור על עצמו עם אותו מפתח. ה-`ciphertext` כולל בתוכו גם את ה-tag לאימות שלמות (authentication tag) בסוף. `self._aesgcm.decrypt()` מבצע בו-זמנית פיענוח ואימות שלמות — אם המידע שונה בדרך, הפונקציה תזרוק חריגה. עטיפת ה-try/except מבטיחה שכשלון פענוח לא יקרוס את השרת.

קטע הקוד הרביעי: **אימות hash לחיצת היד** (`crypto_utils.py`, שורות 35–49)

```python
def verify_handshake_hash(self, challenge: str, response_hash: str) -> bool:
    if not response_hash or len(response_hash) != 64:
        return False
    if not all(c in "0123456789abcdefABCDEF" for c in response_hash):
        return False
    expected = hashlib.sha256(
        (challenge + HANDSHAKE_SECRET).encode("utf-8")
    ).hexdigest()
    try:
        return hmac.compare_digest(
            expected.lower().encode("ascii"),
            response_hash.lower().encode("ascii"),
        )
    except Exception:
        return False
```

הפונקציה מבצעת שני שלבי ולידציה לפני ההשוואה: בדיקה שה-hash הוא בן 64 תווים (SHA-256 hex תמיד 64 תווים), ובדיקה שהוא מכיל רק תווים hex תקינים. לאחר מכן מחושב ה-hash הצפוי ב-`SHA-256(challenge + HANDSHAKE_SECRET)`. ההשוואה מתבצעת דרך `hmac.compare_digest` ולא `==` — ההבדל הקריטי הוא שהשוואה רגילה יכולה לדלוף מידע על ה-hash דרך זמן הביצוע (timing attack), בעוד ש-`compare_digest` תמיד לוקחת אותו זמן ללא קשר לתוכן.

קטע הקוד השישי: **אלגוריתם sliding-window של ה-Rate Limiter** (`rate_limit.py`, שורות 22–28)

```python
def is_blocked(self, key: str) -> bool:
    now = time.monotonic()
    with self._lock:
        q = self._failures[key]
        while q and now - q[0] > self.window_sec:
            q.popleft()
        return len(q) >= self.max_events
```

הלוגיקה מבוססת על `deque` של timestamps. בכל קריאה ל-`is_blocked`, הפונקציה מנקה מהתחלת ה-deque את כל ה-timestamps שישנים מ-`window_sec` שניות (חלון 5 דקות). לאחר הניקוי, אם נותרו `max_events` (12) timestamps או יותר — ה-IP חסום. יתרון הגישה: אין צורך ב-background thread לניקוי; הניקוי קורה בצורה lazy בכל קריאה. `time.monotonic()` נבחר על פני `time.time()` כי הוא אינו מושפע משינויי שעון מערכת.

קטע הקוד השביעי: **broadcast עם snapshot** (`room_registry.py`, שורות 40–48)

```python
def broadcast(self, room: str, payload: dict) -> None:
    body = json.dumps(payload)
    with self._lock:
        targets = list(self._rooms.get(room, set()))
    for client in targets:
        try:
            client.send(body)
        except Exception:
            pass
```

הפונקציה מדגימה דפוס עיצוב חשוב: העתקת snapshot של הסט לפני השליחה. ה-Lock מוחזק רק בזמן ההעתקה ולא בזמן השליחה עצמה — שכן `client.send()` יכולה לקחת זמן אם הלקוח איטי. אם היינו שומרים על ה-Lock בזמן השליחה, היינו חוסמים threads אחרים שרוצים להוסיף או להסיר לקוחות. ה-try/except הבודד לכל לקוח מבטיח שניתוק פתאומי של לקוח אחד לא ימנע שליחת ההודעה לשאר הלקוחות בחדר.

קטע הקוד התשיעי: **חישוב hash לחיצת יד בצד הלקוח** (`chat.js`, שורות 70–77)

```javascript
async function handshakeResponse(challenge, sock) {
  const enc = new TextEncoder();
  const buf = enc.encode(challenge + HANDSHAKE_SECRET);
  const hashBuffer = await crypto.subtle.digest('SHA-256', buf);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  const hashHex = hashArray.map((b) => b.toString(16).padStart(2, '0')).join('');
  sock.send(JSON.stringify({ type: 'auth_response', hash: hashHex }));
}
```

הפונקציה היא המקבילה בצד הלקוח לפונקציית `verify_handshake_hash` בשרת. `TextEncoder.encode()` ממיר את `challenge + HANDSHAKE_SECRET` ל-bytes. `crypto.subtle.digest('SHA-256', buf)` היא ה-Web Crypto API הסטנדרטית — מובנית בדפדפן, מהירה, ואינה דורשת ספריות נוספות. ה-hash שמתקבל הוא `ArrayBuffer` שממופה ל-`Uint8Array` ולאחר מכן לפורמט hex עם `padStart(2, '0')` — חשוב כדי שביטים מובילים עם ערך 0 לא ייחתכו (לדוגמה `0f` ולא `f`).

```javascript
        for (const line of lines) {
          const plaintext = line.trim();
          if (plaintext && !plaintext.includes('CTAP') &&
              socket && socket.readyState === WebSocket.OPEN) {
            socket.send(JSON.stringify({
              type: 'web_msg', text: plaintext, username: 'DEV_' + username
            }));
          }
        }
      }
    } finally {
      reader.releaseLock();
    }
  }
}
```

הפונקציה מיישמת לולאת קריאה אסינכרונית מהפורט הטורי. `serialPort.readable.getReader()` נועל את ה-stream — חשוב לשחרר אותו ב-`finally` כדי לאפשר ניתוק מסודר. `TextDecoder` עם `{ stream: true }` מבצע decode בצורה "נוזלית" — חשוב כי chunk יחיד יכול לחצות באמצע תו multi-byte. הסינון `!plaintext.includes('CTAP')` מסנן שורות מטה-נתונים שה-ESP32 מדפיס בהפעלה (כגון "CTAP FSR 5-Finger Mode Ready"). בדיקת `socket.readyState === WebSocket.OPEN` מבטיחה שלא ננסה לשלוח הודעה כשה-WebSocket מנותק.

# תיאור האלגוריתמים המרכזיים בפרויקט {#תיאור-האלגוריתמים-המרכזיים-בפרויקט}

**סוגייה ראשונה: נטרול רעשים חשמליים ולחיצות כפולות בכפפה**

חיישן FSR הוא רכיב אנלוגי שמוציא ערך בין 0 ל-4095 (12 ביט). הבעיה היא שאפילו ללא לחץ מכוון, קיים רעש חשמלי שגורם לתנודות קטנות בערך. בנוסף, לחיצה אנושית על חיישן ממשיכה לעיתים "לנקוש" (bounce) מספר פעמים לפני שמתייצבת. ללא טיפול, הפירמוור יירשום לחיצה אחת כמספר לחיצות, מה שיגרום לאותיות שגויות.

**אפשרות א' — סינון חומרה (מעגל RC)**

הוספת קבל (capacitor) בטור עם הנגד יוצרת מסנן Low-Pass שחוסם תדרים גבוהים (כלומר שינויים מהירים הנגרמים מרעש ובאונסינג). היתרון הוא שהפתרון פועל ברמת החומרה ואינו מכביד על המעבד. החיסרון הוא שקשה לכייל את ערכי הקבל והנגד במדויק, הפתרון אינו גמיש, ושינוי הסף דורש שינוי פיזי ברכיבים.

**אפשרות ב' — debounce ו-Threshold Hysteresis תוכנתיים**

מימוש בקוד הפירמוור של שני ספים נפרדים: ספסף לחיצה (500) וסף שחרור (300). ספסף הלחיצה גבוה מספיק כדי לסנן רעש, ואילו ספסף השחרור נמוך יותר — מה שיוצר "hysteresis" ומונע רישום לחיצות נוספות בזמן שהחיישן עדיין בתחום הביניים. בנוסף, debounce של 50ms מבטיח שלאחר רישום לחיצה, לא תירשם לחיצה נוספת מאותה אצבע לפחות 50ms.

| פתרון | סינון חומרה (RC) | Debounce + Hysteresis תוכנתי |
| ----: | ----: | ----: |
| יתרונות | אינו מכביד על המעבד; פועל לפני הקריאה | גמיש — ניתן לשנות ספים בקוד; אינו דורש רכיבים נוספים; כיול קל |
| חסרונות | קשה לכיול; דורש שינוי פיזי ברכיבים; לא גמיש | מעביר עומס קטן למעבד; אם הסף שגוי חייב לעדכן firmware |

החלטתי לבחור **באפשרות ב' — debounce ו-Threshold Hysteresis תוכנתיים** מכיוון שהיא מאפשרת לי לכייל ולשנות את הספים בקלות בלי לפרק את הכפפה. בנוסף, העומס על המעבד של ESP32 הוא זניח בהחלט, ועיקר הביצועים הנדרשים הוא בצד השרת ולא בצד החומרה.

---

**סוגייה שנייה: שיטת תעבורת הנתונים מהכפפה לרשת**

לאחר שה-ESP32 בנה אות בינארי, צריך לשלוח אותו לשרת. השאלה היא כיצד — האם לחבר את ה-ESP32 ישירות לרשת Wi-Fi, או לשלוח את הנתונים טורית למחשב המשתמש ולהשאיר לדפדפן לטפל בשליחה לשרת.

**אפשרות א' — ESP32 מחובר ישירות לרשת Wi-Fi**

ה-ESP32 תומך ב-Wi-Fi מובנה. היה ניתן לכתוב בפירמוור חיבור WebSocket ישיר מה-ESP32 לשרת Flask. היתרון הוא שאין תלות בדפדפן וניתן להשתמש בכפפה ללא מחשב. החיסרון הוא שהצפנת AES-256-GCM ב-ESP32 כרוכה בצריכת אנרגיה גבוהה יותר ובתכנות מורכב, ובנוסף ניהול JWT ב-firmware הוא אתגר לא פשוט. בנוסף, חשוב לציין שחיבור Wi-Fi מחייב שמירת סיסמא רשת ב-firmware — בעיית אבטחה מובנית.

**אפשרות ב' — חיבור USB טורי + Web Serial API בדפדפן**

ה-ESP32 שולח הנתונים דרך UART לפורט ה-USB של המחשב. הדפדפן קורא את הנתונים בעזרת Web Serial API ומשלב אותם לתוך חיבור ה-WebSocket הקיים. היתרון הוא שכל ההצפנה, האימות, וניהול ה-JWT מבוצעים בדפדפן — שם הכלים הם בשלים ומאובטחים. ה-firmware נשאר פשוט ולא מכיל מידע רגיש. החיסרון הוא שצריך מחשב עם דפדפן Chrome/Edge לצד הכפפה.

| פתרון | ESP32 ישירות ל-Wi-Fi | USB טורי + Web Serial |
| ----: | ----: | ----: |
| יתרונות | אינו דורש מחשב לצד הכפפה | הצפנה ואימות מתבצעים בדפדפן — כלים בשלים; firmware פשוט ובטוח; אין סיסמאות רשת ב-firmware |
| חסרונות | הצפנה ב-firmware מורכבת; סיסמאות רשת ב-firmware; JWT קשה ל-ESP32 | דורש Chrome/Edge; דורש מחשב לצד הכפפה |

בחרתי **באפשרות ב' — USB טורי + Web Serial API** מכיוון שאבטחה היא עדיפות גבוהה בפרויקט, ואני לא רוצה שסיסמאות רשת יאוחסנו ב-firmware. בנוסף, הצפנת AES-256-GCM ב-JavaScript עם `crypto.subtle` היא אמינה ומתועדת הרבה יותר מאשר מימושה ב-C++ ב-ESP32. הפשטות של ה-firmware — רק קריאה וסריאליזציה — מפחיתה את שטח ההתקפה.

---

**סוגייה שלישית: ניהול יומן אירועים (Audit) ללא פגיעה בביצועי השרת**

כל הודעה, חיבור, וניתוק דורשים רישום ביומן. הבעיה היא שכתיבה ל-SQLite היא פעולה חוסמת (blocking) — בזמן שה-thread כותב, שאר הפעולות ממתינות. אם השרת מטפל בעשרות WebSocket connections, כתיבות תכופות ל-SQLite יכולות לגרום לעיכובים מורגשים.

**אפשרות א' — כתיבה סינכרונית ישירה מהשרת הראשי**

בכל אירוע (חיבור, הודעה, ניתוק) — ה-thread של WebSocket קורא ישירות ל-`log_connection` או `log_message` לפני המשך הטיפול. היתרון הוא הפשטות המלאה — אין threads נוספים, אין תקשורת נוספת. החיסרון הוא שכל כתיבה ל-DB חוסמת את ה-thread הנוכחי, ותחת עומס גבוה יגרמו עיכובים.

**אפשרות ב' — שרת TCP נפרד לניהול ה-Audit**

כל האינטראקציות עם מסד נתוני הבקרה מנותבות דרך שרת TCP עצמאי הפועל ב-thread נפרד. ה-thread הראשי שולח הודעות ל-`AuditDatabase` (שמוגנת ב-Lock), ושרת ה-TCP מאפשר לכלים חיצוניים לשאול ולכתוב ל-DB ללא כל גישה לשרת Flask. היתרון הוא הפרדה מלאה של זרמי עבודה, מה שמאפשר לשרת ה-WebSocket להמשיך לעבד הודעות בזמן שהכתיבה מתבצעת. בנוסף — כלי ניטור חיצוניים יכולים לקרוא ולכתוב ל-DB ישירות דרך ה-TCP.

| פתרון | כתיבה סינכרונית | שרת TCP נפרד |
| ----: | ----: | ----: |
| יתרונות | קוד פשוט מאוד; אין overhead נוסף | אינו חוסם את ה-WebSocket threads; כלים חיצוניים יכולים לגשת ל-DB; הפרדה ברורה בין אחריות |
| חסרונות | תחת עומס — עיכובי כתיבה פוגעים בתגובתיות הצ'אט | מורכבות גבוהה יותר; thread נוסף לנהל |

בחרתי **בשרת TCP נפרד** מכיוון שמבחינת ארכיטקטורה זה הפתרון הנקי יותר. הפרדת אחריות (Separation of Concerns) היא עיקרון עיצוב חשוב, ויצירת שרת TCP עצמאי לטיפול ב-Audit מאפשרת לי גם לפתח כלי ניטור חיצוניים בקלות בעתיד, ללא שינוי בשרת הראשי.

# תיאור מסכי הפרויקט {#תיאור-מסכי-הפרויקט}

**תיאור כל מסך**

* **מסך נחיתה (Landing)** — המסך הראשון שהמשתמש רואה עם כניסה לאתר. מציג את שם המערכת CTAP Chat, תיאור קצר ("Secure Hardware & Web Chat System"), וכפתור Log In המוביל למסך הכניסה. עיצוב מינימליסטי עם רקע כהה.

* **מסך כניסה (Login)** — טופס כניסה עם שני שדות: שם משתמש וסיסמא. לחיצה על Submit שולחת `POST /login`; תשובה עם JWT שמורה ב-`localStorage`. הניתוב מ-Login מתבצע אוטומטית לדף Chat לאחר כניסה מוצלחת. מוצגת הודעת שגיאה ברורה אם הכניסה נכשלה.

* **מסך הרשמה (Register)** — טופס הרשמה עם שדות שם משתמש וסיסמא. שולח `POST /register`; בהצלחה — מנתב אוטומטית למסך הכניסה. מוצגת הודעת שגיאה אם שם המשתמש תפוס או אם הסיסמא קצרה מדי.

* **מסך חדר השיחה (Chat)** — המסך המרכזי של המערכת. מחולק לשתי חלוניות:
  * **Sidebar שמאלית** — ניהול חדרים (שדה + כפתור Join), הצגת החדר הנוכחי, חיבור לכפפת ESP32 (כפתור Connect/Disconnect + מחוון מצב), ומחוון מצב חיבור לשרת.
  * **אזור הודעות** — רשימה גוללת של הודעות הצ'אט, שדה קלט, וכפתור Send. הודעות המשתמש הנוכחי מוצגות בצד ימין (own), הודעות משתמשים אחרים בצד שמאל.

* **מסך יומני בקרה (Audit)** — מסך הנגיש למשתמשים מחוברים בלבד. מציג שתי טבלאות: יומן חיבורים (עמודות: client_address, event_type, room, timestamp) ויומן הודעות (עמודות: msg_id, sender_address, room, msg_hash, msg_type, timestamp). נתונים נשלפים מ-`GET /audit/logs` ו-`GET /audit/connections` עם JWT בלבד.

**תרשים זרימה של מעבר בין המסכים**

```
[Landing]
    |
    | לחיצה על "Log In"
    v
[Login] <----> [Register]
    |               |
    | JWT בידינו    |
    v               v
[Chat] ---------> [Audit]
    |
    | Logout
    v
[Landing]
```

# פרוטוקול התקשורת {#פרוטוקול-התקשורת}

**פרוטוקול WebSocket — מחזור חיי חיבור**

| סוג הודעה | תיאור | סוג/שדה | ממי אל מי | דוגמה |
| ----: | ----: | ----: | ----: | ----: |
| **auth_challenge** | שרת שולח challenge אקראי עם כל חיבור חדש | `type`, `challenge` (32 hex) | שרת → לקוח | `{"type":"auth_challenge","challenge":"a1b2c3..."}` |
| **auth_response** | לקוח שולח SHA-256(challenge + secret) | `type`, `hash` (64 hex) | לקוח → שרת | `{"type":"auth_response","hash":"e3b0c4..."}` |
| **auth_result** | תוצאת האימות | `type`, `status` (OK/FAIL), `message` | שרת → לקוח | `{"type":"auth_result","status":"OK","message":"Authentication successful."}` |
| **join_room** | בקשת הצטרפות לחדר | `type`, `room` (string, עד 64 תווים) | לקוח → שרת | `{"type":"join_room","room":"team-alpha"}` |
| **room_joined** | אישור הצטרפות לחדר | `type`, `room` | שרת → לקוח | `{"type":"room_joined","room":"team-alpha"}` |
| **web_msg** | הודעת טקסט רגילה מהדפדפן | `type`, `text`, `username` | לקוח → שרת | `{"type":"web_msg","text":"hello","username":"alice"}` |
| **CTAP_MSG** | הודעה מוצפנת AES-256-GCM מהכפפה | `type`, `msg_id`, `timestamp`, `payload` {nonce, ciphertext} | לקוח → שרת | `{"type":"CTAP_MSG","payload":{"nonce":"base64","ciphertext":"base64"}}` |
| **chat_message** | הודעה מופצת לכל מחוברי החדר | `type`, `text`, `timestamp`, `msg_id`, `sender`, `room` | שרת → כל הלקוחות בחדר | `{"type":"chat_message","text":"hi","sender":"alice","room":"default"}` |

**פרוטוקול REST — ניהול משתמשים ויומן**

| נקודת קצה | מתודה | גוף הבקשה | תגובה מוצלחת | הגנה |
| ----: | ----: | ----: | ----: | ----: |
| `/register` | POST | `{"username":"...","password":"..."}` | `{"message":"User registered successfully"}` | ולידציה (3-32 אותיות, 8-128 תווים לסיסמא) |
| `/login` | POST | `{"username":"...","password":"..."}` | `{"token":"JWT","username":"..."}` | Rate Limit; bcrypt verify |
| `/audit/logs` | GET | — | `[{msg rows}]` | JWT Bearer |
| `/audit/connections` | GET | — | `[{conn rows}]` | JWT Bearer |

**פרוטוקול TCP Audit — פקודות מנהל**

| פקודה (op) | תיאור | שדות נדרשים | דוגמה לתגובה |
| ----: | ----: | ----: | ----: |
| AUTH (שורה ראשונה) | אימות בסוד משותף | `AUTH <secret>` (plain text) | `{"ok":true,"message":"ready"}` |
| `get_logs` | שליפת הודעות אחרונות | `limit` (1-500, ברירת מחדל 100) | `{"ok":true,"data":[...]}` |
| `get_connections` | שליפת חיבורים אחרונים | `limit` | `{"ok":true,"data":[...]}` |
| `insert_connection` | הכנסת שורת חיבור | `client_address`, `event_type`, `room` | `{"ok":true,"inserted":"connection"}` |
| `insert_message` | הכנסת שורת הודעה | `msg_id`, `sender_address`, `msg_hash` | `{"ok":true,"inserted":"message"}` |
| `quit` | סיום session | — | `{"ok":true,"message":"bye"}` |

# פרק ז' – מדריך למשתמש {#פרק-ז-מדריך-למשתמש}

# הדרכה למשתמש ה-Web {#הדרכה-למשתמש-ה-web}

**שלב 1 — הרשמה**

פתחו את הדפדפן (Chrome או Edge מומלצים) ועברו לכתובת `http://localhost:8766` או לכתובת השרת שקיבלתם. לחצו על הקישור "Register" ומלאו שם משתמש (3 עד 32 תווים, אותיות ומספרים בלבד) וסיסמא (לפחות 8 תווים). לחצו Submit — אם ההרשמה הצליחה, תועברו אוטומטית לדף הכניסה.

**שלב 2 — כניסה**

מלאו שם משתמש וסיסמא ולחצו Submit. לאחר כניסה מוצלחת, תועברו לדף הצ'אט. שימו לב: לאחר 12 ניסיונות כניסה כושלים רצופים, ה-IP שלכם ייחסם למשך 5 דקות.

**שלב 3 — שיחה בחדר**

בפתיחת דף הצ'אט אתם מצורפים אוטומטית לחדר `default`. כדי להצטרף לחדר אחר, הקלידו שם חדר בשדה שבסרגל הצד הימני ולחצו Join. הודעות החדר הנוכחי יוצגו באזור המרכזי. כדי לשלוח הודעה, הקלידו בשדה הטקסט בתחתית המסך ולחצו Send או הקישו Enter.

**שלב 4 — יציאה**

לחצו על כפתור Logout בפינה העליונה — ה-token יימחק ותועברו לדף הנחיתה.

# הדרכה למפעיל החומרה (הכפפה) {#הדרכה-למפעיל-החומרה}

**שלב 1 — הכנת החומרה**

חברו את ה-ESP32 למחשב בכבל USB. פתחו Chrome ועברו לדף הצ'אט. הבטיחו שהלוח ניצת (נורית ה-LED על ה-ESP32 תדלוק).

**שלב 2 — חיבור ה-Serial**

בסרגל הצד של דף הצ'אט, לחצו על כפתור "Connect ESP32". ייפתח חלון בחירה של הדפדפן המציג את הפורטים הזמינים — בחרו את הפורט המתאים ל-ESP32 שלכם (בדרך כלל COM3 או COM4 ב-Windows). לאחר הבחירה, המצב יעבור ל-`State: Connected`.

**שלב 3 — הקלדה בכפפה**

| שילוב אצבעות | אות |
| ----: | ----: |
| 00001 (קמיצה בלבד) | a |
| 00010 (אמה בלבד) | b |
| 00011 (קמיצה + אמה) | c |
| 00100 (אמה + ... ) | d |
| ... | ... |
| 11010 (כל האצבעות פרט לקמיצה) | z |
| 11111 (כל האצבעות — פעם אחת) | רווח |
| 11111 (כל האצבעות — פעמיים ברצף) | שליחת ההודעה |

כדי לשלוח הודעה: הקלידו את האותיות בזו אחר זו (כל אות — לחיצת שילוב ברצף תוך 300ms), הוסיפו רווח בין מילים בלחיצת כל חמש האצבעות פעם אחת, ושלחו את ההודעה בלחיצת כל חמש האצבעות פעמיים ברצף מהיר.

**שלב 4 — ניתוק**

```cpp
// ===== ESP32 PIN CONFIG =====
// Fingers: 34, 35, 32, 33, 25
int fsrPins[] = {34, 35, 32, 33, 25};
int numFSRs = 5;

int ledPin = 2;

// ===== SETTINGS =====
int pressThreshold = 500;
int releaseThreshold = 300;
unsigned long debounceTime = 50;
unsigned long letterWait = 300;

// ===== STATE =====
bool fingerCurrentlyPressed[5] = {false};
bool fingerHit[5] = {false};
unsigned long firstHitTime = 0;
unsigned long lastDebounce[5] = {0};

int allFingersCount = 0;
String textSentence = "";

void setup() {
  Serial.begin(115200);
  pinMode(ledPin, OUTPUT);
  analogReadResolution(12);
  analogSetAttenuation(ADC_11db);
  Serial.println("CTAP FSR 5-Finger Mode Ready");
  Serial.println("Pins: 34, 35, 32, 33, 25 (Fingers)");
}

void loop() {
  unsigned long now = millis();

  for (int i = 0; i < numFSRs; i++) {
    int reading = analogRead(fsrPins[i]);
    if (!fingerCurrentlyPressed[i] &&
        reading > pressThreshold &&
        (now - lastDebounce[i] > debounceTime)) {
      lastDebounce[i] = now;
      fingerCurrentlyPressed[i] = true;
      digitalWrite(ledPin, !digitalRead(ledPin));
      fingerHit[i] = true;
      if (firstHitTime == 0) firstHitTime = now;
    }
    if (fingerCurrentlyPressed[i] && reading < releaseThreshold) {
      fingerCurrentlyPressed[i] = false;
    }
  }

  if (firstHitTime != 0 && (now - firstHitTime > letterWait)) {
    String currentBits = "";
    for (int i = 0; i < 5; i++) {
      currentBits += fingerHit[i] ? "1" : "0";
    }
    if (currentBits == "11111") {
      allFingersCount++;
      if (allFingersCount >= 2) {
        textSentence.trim();
        Serial.println(textSentence);
        textSentence = "";
        allFingersCount = 0;
        digitalWrite(ledPin, HIGH); delay(200); digitalWrite(ledPin, LOW);
      } else {
        if (textSentence.length() > 0 &&
            textSentence[textSentence.length() - 1] != ' ') {
          textSentence += " ";
        }
      }
    } else {
      allFingersCount = 0;
      String letter = getLetterFromBits(currentBits);
      if (letter != "") textSentence += letter;
    }
    resetFingers();
  }
  delay(5);
}

String getLetterFromBits(String bits) {
  if (bits == "00001") return "a";
  if (bits == "00010") return "b";
  if (bits == "00011") return "c";
  if (bits == "00100") return "d";
  if (bits == "00101") return "e";
  if (bits == "00110") return "f";
  if (bits == "00111") return "g";
  if (bits == "01000") return "h";
  if (bits == "01001") return "i";
  if (bits == "01010") return "j";
  if (bits == "01011") return "k";
  if (bits == "01100") return "l";
  if (bits == "01101") return "m";
  if (bits == "01110") return "n";
  if (bits == "01111") return "o";
  if (bits == "10000") return "p";
  if (bits == "10001") return "q";
  if (bits == "10010") return "r";
  if (bits == "10011") return "s";
  if (bits == "10100") return "t";
  if (bits == "10101") return "u";
  if (bits == "10110") return "v";
  if (bits == "10111") return "w";
  if (bits == "11000") return "x";
  if (bits == "11001") return "y";
  if (bits == "11010") return "z";
  return "";
}

  for (int i = 0; i < 5; i++) fingerHit[i] = false;
  firstHitTime = 0;
}
```

### backend/crypto_utils.py

```python
"""AES-GCM and handshake helpers; OOP facade CTAPCrypto."""

import base64
import binascii
import hashlib
import hmac
import os

from cryptography.hazmat.primitives.ciphers.aead import AESGCM

FIXED_AES_KEY_HEX = (
    "000102030405060708090a0b0c0d0e0f101112131415161718191a1b1c1d1e1f"
)
HANDSHAKE_SECRET = "CTAP-GLOVE-AUTH-2026"


class CTAPCrypto:
    """Symmetric crypto + handshake verification for CTAP wire format."""

    __slots__ = ("_aesgcm",)

    def __init__(self, key_hex: str = FIXED_AES_KEY_HEX) -> None:
        key = binascii.unhexlify(key_hex)
        self._aesgcm = AESGCM(key)

    def decrypt_payload(self, payload: dict) -> str:
        try:
            nonce = base64.b64decode(payload["nonce"])
            ciphertext = base64.b64decode(payload["ciphertext"])
            plaintext_bytes = self._aesgcm.decrypt(nonce, ciphertext, None)
            return plaintext_bytes.decode("utf-8")
        except Exception as e:
            return f"[DECRYPTION FAILED] {e}"

    def verify_handshake_hash(self, challenge: str, response_hash: str) -> bool:
        if not response_hash or len(response_hash) != 64:
            return False
        if not all(c in "0123456789abcdefABCDEF" for c in response_hash):
            return False
        expected = hashlib.sha256(
            (challenge + HANDSHAKE_SECRET).encode("utf-8")
        ).hexdigest()
        try:
            return hmac.compare_digest(
                expected.lower().encode("ascii"),
                response_hash.lower().encode("ascii"),
            )
        except Exception:
            return False

    @staticmethod
    def generate_challenge() -> str:
        return os.urandom(16).hex()

    @staticmethod
    def generate_hash(content: str) -> str:
```

```python
"""SQLite audit DB and users; matches legacy schema. OOP facade: AuditDatabase."""

from __future__ import annotations

import json
import sqlite3
import threading
from datetime import datetime
from pathlib import Path
from typing import Any

BACKEND_DIR = Path(__file__).resolve().parent
DB_PATH = str(BACKEND_DIR / "ctap_audit.db")
DATA_DIR = BACKEND_DIR / "data"
RUNTIME_META_PATH = DATA_DIR / "server_runtime.json"


class AuditDatabase:
    """Thread-safe SQLite access for users and audit tables."""

    __slots__ = ("_conn", "_lock", "_path")

    def __init__(self, path: str) -> None:
        self._path = path
        self._lock = threading.Lock()
        self._conn: sqlite3.Connection | None = None

    def _connection(self) -> sqlite3.Connection:
        if self._conn is None:
            self._conn = sqlite3.connect(self._path, check_same_thread=False)
            self._conn.row_factory = sqlite3.Row
        return self._conn

    def get_sqlite_connection(self) -> sqlite3.Connection:
        return self._connection()

    def init_schema(self) -> sqlite3.Connection:
        with self._lock:
            conn = self._connection()
            cur = conn.cursor()
            cur.execute("""
                CREATE TABLE IF NOT EXISTS connections (
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    client_address TEXT NOT NULL,
                    event_type TEXT NOT NULL CHECK(event_type IN
                        ('CONNECT', 'DISCONNECT', 'AUTH_SUCCESS', 'AUTH_FAIL')),
                    room TEXT DEFAULT 'default',
                    timestamp TEXT NOT NULL
                )
            """)
            cur.execute("""
                CREATE TABLE IF NOT EXISTS messages (
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    msg_id TEXT UNIQUE NOT NULL,
                    sender_address TEXT NOT NULL,
                    room TEXT NOT NULL,
                    msg_hash TEXT NOT NULL,
                    msg_type TEXT NOT NULL,
                    timestamp TEXT NOT NULL
                )
            """)
            cur.execute("""
                CREATE TABLE IF NOT EXISTS users (
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    username TEXT UNIQUE NOT NULL,
                    password TEXT NOT NULL
                )
            """)
            conn.commit()
            print("[DB] Connected to SQLite database.")
        return self._connection()

    @staticmethod
    def _row_to_dict(row: sqlite3.Row) -> dict[str, Any]:
        return {k: row[k] for k in row.keys()}

    def log_connection(self, client_address: str, event_type: str, room: str = "default") -> None:
        with self._lock:
            conn = self._connection()
            conn.execute(
                "INSERT INTO connections (client_address, event_type, room, timestamp) "
                "VALUES (?, ?, ?, ?)",
                (str(client_address), event_type, room, datetime.now().isoformat()),
            )
            conn.commit()

    def log_message(self, msg_id: str, sender_address: str, room: str,
                    msg_hash: str, msg_type: str) -> None:
        with self._lock:
            conn = self._connection()
            conn.execute(
                "INSERT OR IGNORE INTO messages "
                "(msg_id, sender_address, room, msg_hash, msg_type, timestamp) "
                "VALUES (?, ?, ?, ?, ?, ?)",
                (msg_id, str(sender_address), room, msg_hash, msg_type,
                 datetime.now().isoformat()),
            )
            conn.commit()

    def create_user(self, username: str, password_hash: str) -> None:
        with self._lock:
            conn = self._connection()
            conn.execute("INSERT INTO users (username, password) VALUES (?, ?)",
                         (username, password_hash))
            conn.commit()

    def get_user_by_username(self, username: str) -> dict[str, Any] | None:
        with self._lock:
            conn = self._connection()
            cur = conn.execute("SELECT * FROM users WHERE username = ?", (username,))
            row = cur.fetchone()
        return self._row_to_dict(row) if row else None

    def fetch_audit_messages(self, limit: int = 100) -> list[dict[str, Any]]:
        with self._lock:
            conn = self._connection()
            cur = conn.execute(
                "SELECT * FROM messages ORDER BY timestamp DESC LIMIT ?", (limit,))
            rows = cur.fetchall()
        return [self._row_to_dict(r) for r in rows]

    def fetch_audit_connections(self, limit: int = 100) -> list[dict[str, Any]]:
        with self._lock:
            conn = self._connection()
            cur = conn.execute(
                "SELECT * FROM connections ORDER BY timestamp DESC LIMIT ?", (limit,))
            rows = cur.fetchall()
        return [self._row_to_dict(r) for r in rows]

    def checkpoint_wal(self) -> None:
        with self._lock:
            conn = self._connection()
            try:
                conn.execute("PRAGMA wal_checkpoint(PASSIVE)")
            except sqlite3.Error:
                pass
            conn.commit()

    def write_runtime_meta_json(self, uptime_sec: float) -> None:
        DATA_DIR.mkdir(parents=True, exist_ok=True)
        meta = {"uptime_sec": round(uptime_sec, 3), "database_path": self._path,
                "pid": __import__("os").getpid(), "updated_iso": datetime.now().isoformat()}
        tmp = RUNTIME_META_PATH.with_suffix(".tmp")
        tmp.write_text(json.dumps(meta, indent=2), encoding="utf-8")
        tmp.replace(RUNTIME_META_PATH)
```

```python
"""Simple sliding-window rate limiter for brute-force mitigation (login)."""

from __future__ import annotations
import threading
import time
from collections import defaultdict, deque
from typing import Deque, DefaultDict


class LoginRateLimiter:
    """Tracks failed login timestamps per IP; blocks when threshold exceeded."""

    __slots__ = ("_lock", "_failures", "max_events", "window_sec")

    def __init__(self, max_events: int = 12, window_sec: float = 300.0) -> None:
        self._lock = threading.Lock()
        self._failures: DefaultDict[str, Deque[float]] = defaultdict(deque)
        self.max_events = max_events
        self.window_sec = window_sec

    def is_blocked(self, key: str) -> bool:
        now = time.monotonic()
        with self._lock:
            q = self._failures[key]
            while q and now - q[0] > self.window_sec:
                q.popleft()
            return len(q) >= self.max_events

    def record_failure(self, key: str) -> None:
        now = time.monotonic()
        with self._lock:
            q = self._failures[key]
            q.append(now)
            while q and now - q[0] > self.window_sec:
                q.popleft()

    def clear(self, key: str) -> None:
        with self._lock:
            self._failures.pop(key, None)
```

### backend/room_registry.py

```python
"""Thread-safe registry of WebSocket clients per chat room."""

from __future__ import annotations
import json
import threading
from typing import Any


class RoomRegistry:
    """Maps room names to sets of WebSocket connections."""

    __slots__ = ("_lock", "_rooms")

    def __init__(self) -> None:
        self._lock = threading.Lock()
        self._rooms: dict[str, set[Any]] = {}

    def add(self, room: str, ws: Any) -> None:
        with self._lock:
            if room not in self._rooms:
                self._rooms[room] = set()
            self._rooms[room].add(ws)

    def discard(self, room: str, ws: Any) -> None:
        with self._lock:
            bucket = self._rooms.get(room)
            if bucket and ws in bucket:
                bucket.discard(ws)

    def move(self, ws: Any, old_room: str, new_room: str) -> None:
        with self._lock:
            old = self._rooms.get(old_room)
            if old and ws in old:
                old.discard(ws)
            if new_room not in self._rooms:
                self._rooms[new_room] = set()
            self._rooms[new_room].add(ws)

    def broadcast(self, room: str, payload: dict) -> None:
        body = json.dumps(payload)
        with self._lock:
            targets = list(self._rooms.get(room, set()))
        for client in targets:
            try:
                client.send(body)
            except Exception:
                pass

    def room_count(self) -> int:
        with self._lock:
            return len(self._rooms)
```

### backend/tcp_db_socket.py

```python
"""
Plain TCP socket (not WebSocket) for authenticated audit DB reads and updates.

Uses the socket module. Each client connects, sends AUTH <token>,
then newline-delimited JSON commands (one response line per command).

Environment:
  CTAP_TCP_DB_PORT      — listen port (default 8767)
  CTAP_TCP_DB_BIND      — bind address (default 0.0.0.0)
  CTAP_TCP_ADMIN_SECRET — shared secret for AUTH line (change in production)
  CTAP_TCP_DB_ENABLED   — set to 0 to disable the TCP listener
"""

from __future__ import annotations

import hashlib
import hmac
import json
import os
import socket
import threading
from typing import Any

from backend.database import AuditDatabase

_ALLOWED_EVT = frozenset({"CONNECT", "DISCONNECT", "AUTH_SUCCESS", "AUTH_FAIL"})
_default_secret = "CTAP-TCP-ADMIN-DEV-CHANGE-ME"


def _auth_ok(provided: str, expected: str) -> bool:
    """Compare secrets without leaking length via SHA-256 digest compare."""
    return hmac.compare_digest(
        hashlib.sha256(provided.encode("utf-8")).digest(),
        hashlib.sha256(expected.encode("utf-8")).digest(),
    )


class TcpAuditDbSocketServer(threading.Thread):
    """Thread hosting SOCK_STREAM TCP accepts; each client handled in its own thread."""

    __slots__ = ("_audit", "_bind", "_port", "_secret", "_stop", "_sock")

    def __init__(self, audit: AuditDatabase, bind: str, port: int, secret: str) -> None:
        super().__init__(daemon=True, name="CTAP-TcpAuditDbSocket")
        self._audit = audit
        self._bind = bind
        self._port = port
        self._secret = secret
        self._stop = threading.Event()
        self._sock: socket.socket | None = None

    def stop(self) -> None:
        self._stop.set()
        if self._sock is not None:
            try:
                self._sock.close()
            except OSError:
                pass

    def run(self) -> None:
        srv = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        srv.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        self._sock = srv
        try:
            srv.bind((self._bind, self._port))
            srv.listen(16)
            srv.settimeout(1.0)
        except OSError as e:
            print(f"[TCP-DB] Failed to bind {self._bind}:{self._port} — {e}")
            return

        print(f"[TCP-DB] Audit socket listening on {self._bind}:{self._port} (SOCK_STREAM)")

        while not self._stop.is_set():
            try:
                conn, addr = srv.accept()
            except socket.timeout:
                continue
            except OSError:
                break
            t = threading.Thread(
                target=self._client_session,
                args=(conn, addr),
                daemon=True,
                name=f"TcpDbClient-{addr[0]}:{addr[1]}",
            )
            t.start()

    def _send(self, conn: socket.socket, obj: dict[str, Any]) -> None:
        line = json.dumps(obj, ensure_ascii=False) + "\n"
        conn.sendall(line.encode("utf-8"))

    def _client_session(self, conn: socket.socket, addr: tuple[Any, ...]) -> None:
        peer = f"{addr[0]}:{addr[1]}"
        try:
            buf = b""
            while b"\n" not in buf and not self._stop.is_set():
                chunk = conn.recv(4096)
                if not chunk:
                    return
                buf += chunk
                if len(buf) > 65536:
                    self._send(conn, {"ok": False, "error": "line too long"})
                    return

            line, _, rest = buf.partition(b"\n")
            first = line.decode("utf-8", errors="replace").strip()
            if not first.upper().startswith("AUTH "):
                self._send(conn, {"ok": False, "error": "first line must be: AUTH <token>"})
                return
            token = first[5:].strip()
            if not _auth_ok(token, self._secret):
                self._send(conn, {"ok": False, "error": "authentication failed"})
                return

            self._send(conn, {"ok": True, "message": "ready", "peer": peer})

            buffer = rest
            while True:
                while b"\n" not in buffer:
                    chunk = conn.recv(65536)
                    if not chunk:
                        return
                    buffer += chunk
                    if len(buffer) > 1_048_576:
                        self._send(conn, {"ok": False, "error": "request too large"})
                        return
                raw_line, _, buffer = buffer.partition(b"\n")
                cmd_text = raw_line.decode("utf-8", errors="replace").strip()
                if not cmd_text:
                    continue
                try:
                    cmd = json.loads(cmd_text)
                except json.JSONDecodeError as e:
                    self._send(conn, {"ok": False, "error": f"invalid json: {e}"})
                    continue

                ok, payload = self._run_command(cmd)
                if not ok and payload == "bye":
                    self._send(conn, {"ok": True, "message": "bye"})
                    return
                if ok:
                    self._send(conn, {"ok": True, **payload})
                else:
                    self._send(conn, {"ok": False, "error": str(payload)})
        except (OSError, ConnectionError) as e:
            print(f"[TCP-DB] Client {peer} error: {e}")
        finally:
            try:
                conn.close()
            except OSError:
                pass
            print(f"[TCP-DB] Client {peer} disconnected")

    def _run_command(self, cmd: dict[str, Any]) -> tuple[bool, str | dict[str, Any]]:
        op = cmd.get("op")
        if op == "quit":
            return False, "bye"

        if op == "get_logs":
            try:
                limit = int(cmd.get("limit", 100))
            except (TypeError, ValueError):
                limit = 100
            limit = max(1, min(limit, 500))
            rows = self._audit.fetch_audit_messages(limit)
            return True, {"data": rows}

        if op == "get_connections":
            try:
                limit = int(cmd.get("limit", 100))
            except (TypeError, ValueError):
                limit = 100
            limit = max(1, min(limit, 500))
            rows = self._audit.fetch_audit_connections(limit)
            return True, {"data": rows}

        if op == "insert_connection":
            client_address = str(cmd.get("client_address", "")).strip()
            event_type = str(cmd.get("event_type", "")).strip()
            room = str(cmd.get("room", "default")).strip() or "default"
            if not client_address or event_type not in _ALLOWED_EVT:
                return (
                    False,
                    "client_address required; event_type must be "
                    "CONNECT|DISCONNECT|AUTH_SUCCESS|AUTH_FAIL",
                )
            if len(room) > 128:
                room = room[:128]
            self._audit.log_connection(client_address, event_type, room)
            return True, {"inserted": "connection"}

        if op == "insert_message":
            msg_id = str(cmd.get("msg_id", "")).strip()
            sender = str(cmd.get("sender_address", "")).strip()
            room = str(cmd.get("room", "default")).strip() or "default"
            msg_hash = str(cmd.get("msg_hash", "")).strip()
            msg_type = str(cmd.get("msg_type", "tcp_import")).strip() or "tcp_import"
            if not msg_id or not sender or not msg_hash:
                return False, "msg_id, sender_address, msg_hash required"
            if len(room) > 128:
                room = room[:128]
            if len(msg_type) > 32:
                msg_type = msg_type[:32]
            self._audit.log_message(msg_id, sender, room, msg_hash, msg_type)
            return True, {"inserted": "message"}

        return (
            False,
            f"unknown op: {op!r}; use get_logs, get_connections, "
            "insert_connection, insert_message, quit",
        )


_tcp_server: TcpAuditDbSocketServer | None = None
_tcp_lock = threading.Lock()


def start_tcp_db_socket_server(audit: AuditDatabase) -> TcpAuditDbSocketServer | None:
    """Start the TCP DB socket once (idempotent). Returns None if disabled."""
    global _tcp_server
    if os.environ.get("CTAP_TCP_DB_ENABLED", "1").strip().lower() in ("0", "false", "no"):
        print("[TCP-DB] Disabled (CTAP_TCP_DB_ENABLED=0)")
        return None
    with _tcp_lock:
        if _tcp_server is not None and _tcp_server.is_alive():
            return _tcp_server
        bind = os.environ.get("CTAP_TCP_DB_BIND", "0.0.0.0").strip()
        port = int(os.environ.get("CTAP_TCP_DB_PORT", "8767"))
        secret = os.environ.get("CTAP_TCP_ADMIN_SECRET", _default_secret)
        _tcp_server = TcpAuditDbSocketServer(audit, bind, port, secret)
        _tcp_server.start()
        return _tcp_server
```

```javascript
(function () {
  const token = localStorage.getItem('token');
  const username = localStorage.getItem('username') || '';
  document.getElementById('audit-user').textContent = 'User: ' + username;
  document.getElementById('audit-logout').addEventListener('click', () => {
    localStorage.removeItem('token');
    localStorage.removeItem('username');
    location.href = '/';
  });

  let activeTab = 'messages';
  const loading = document.getElementById('audit-loading');
  const wrap = document.getElementById('audit-table-wrap');
  const tabMsg = document.getElementById('tab-messages');
  const tabConn = document.getElementById('tab-connections');

  function setTabs() {
    tabMsg.classList.toggle('active', activeTab === 'messages');
    tabConn.classList.toggle('active', activeTab === 'connections');
  }

  function esc(s) {
    if (s == null) return '';
    const d = document.createElement('div');
    d.textContent = s;
    return d.innerHTML;
  }

  async function fetchLogs() {
    loading.style.display = 'block';
    wrap.style.display = 'none';
    loading.textContent = '[ Loading logs... ]';
    setTabs();
    try {
      const path = activeTab === 'messages' ? '/audit/logs' : '/audit/connections';
      const res = await fetch(path, {
        headers: { Authorization: 'Bearer ' + token },
      });
      if (res.status === 401) {
        location.href = '/login';
        return;
      }
      const rows = await res.json();
      let thead;
      if (activeTab === 'messages') {
        thead =
          '<tr><th>#</th><th>Time</th><th>Username</th><th>Room</th>' +
          '<th>Type</th><th>SHA256 Fingerprint</th></tr>';
        wrap.innerHTML =
          '<table class="audit-table"><thead>' + thead + '</thead><tbody>' +
          rows.map((m) =>
            '<tr><td style="color:var(--text-secondary)">' + esc(m.id) + '</td><td>' +
            esc(new Date(m.timestamp).toLocaleString()) + '</td><td>' +
            esc(m.sender_address) + '</td><td>' + esc(m.room) + '</td><td>' +
            esc(m.msg_type) + '</td><td style="font-size:11px;opacity:0.6">' +
            esc(m.msg_hash) + '</td></tr>'
          ).join('') + '</tbody></table>';
      } else {
        thead = '<tr><th>#</th><th>Time</th><th>IP Address</th><th>Event</th><th>Room</th></tr>';
        wrap.innerHTML =
          '<table class="audit-table"><thead>' + thead + '</thead><tbody>' +
          rows.map((c) => {
            const ev = String(c.event_type || '');
            const col = ev.includes('SUCCESS')
              ? 'var(--success)'
              : ev.includes('FAIL')
                ? 'var(--error)'
                : 'inherit';
            return (
              '<tr><td style="color:var(--text-secondary)">' + esc(c.id) + '</td><td>' +
              esc(new Date(c.timestamp).toLocaleString()) + '</td><td>' +
              esc(c.client_address) + '</td><td><span style="color:' + col + '">[' +
              esc(ev) + ']</span></td><td>' + esc(c.room) + '</td></tr>'
            );
          }).join('') + '</tbody></table>';
      }
    } catch {
      loading.textContent = '[ Failed to load ]';
      return;
    }
    loading.style.display = 'none';
    wrap.style.display = 'block';
  }

  tabMsg.addEventListener('click', () => { activeTab = 'messages'; fetchLogs(); });
  tabConn.addEventListener('click', () => { activeTab = 'connections'; fetchLogs(); });

  fetchLogs();
})();
```
